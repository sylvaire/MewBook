# 需求：统计页支出构成分类下钻

## 背景

统计页「支出构成」当前展示各分类支出金额与占比（默认 Top 5），**不可点击**，用户无法从汇总进入明细。

## 目标用户价值

用户看到某分类占比偏高时，需要快速查看该分类在**同一统计时间范围**内有哪些具体账单，无需回到首页按月翻找。

## 功能需求

### FR-1 入口与范围

- 在统计页的「支出构成」列表中，**每一行分类**可点击（含「未知」分类，若有支出）。
- 下钻后展示的支出记录须与统计页当前选择一致：
  - 时间粒度：`TimeRange`（周 / 月 / 年）；
  - 当前锚点周期（与 `StatisticsViewModel` 中 `anchorDate` + `getDateRange` 一致）；
  - 默认账本（与统计页当前使用的流水过滤逻辑一致）。

### FR-2 明细内容

- 列表仅展示**支出**类型（`RecordType.EXPENSE`）、且 `categoryId` 等于所选分类的记录。
- 单条展示信息至少与首页流水项对齐（金额、日期、备注等现有 `RecordItem` 能力）；只读即可，不要求在本页编辑（编辑可作为后续增强）。

### FR-3 导航与返回

- 从统计进入子页后，提供**返回**到统计页（系统返回键与顶栏返回均可接受）。
- 子页标题建议包含：**分类名称** + **当前周期文案**（与统计顶栏周期描述一致或简化版）。

### FR-4 空状态

- 若该分类在周期内无支出记录（边界情况），展示友好空状态文案。

## 非目标（本里程碑不做）

- 不修改统计图表算法与「支出构成」Top 5 截断规则（除非为实现点击必须微调布局）。
- 不在此里程碑实现跨分类筛选或导出。

## 验收标准（UAT）

1. 统计页切换周/月/年并切换周期后，点击某分类，明细页记录日期均在当前统计周期内。
2. 明细页仅含该分类支出，金额合计与统计页该分类汇总一致（允许四舍五入展示差异说明）。
3. 返回统计页后，统计状态未被意外重置。

## 依赖与假设

- 沿用现有 `Record` / `RecordRepository` 或 `GetRecordsUseCase`，按 `ledgerId` + 日期区间 + 分类 + 类型过滤；若现有 API 不足，在数据层增加查询方法。
