# 需求：网盘自动备份

## 背景

用户已经可以在 DAV 设置页手动“导出到 DAV”，但需要记得主动操作。新增自动备份后，用户只要开启开关，App 每个本地日期首次进入前台时自动将完整 JSON 备份上传到 WebDAV。

## 目标用户价值

- 降低忘记备份导致的数据丢失风险。
- 保持现有 DAV 配置与手动导出/导入流程，不引入额外后台任务或复杂调度。
- 自动清理旧自动备份，避免长期使用后远端目录无限增长，同时不误删手动导出的保留点。

## 功能需求

### FR-1 开关与配置

- DAV 设置页新增“打开 App 自动备份”开关。
- 复用 `DavConfig.isEnabled` 保存开关状态；用户点击“保存配置”后生效。
- 未配置服务器地址或用户名时，即使开关为开启也不执行自动备份。

### FR-2 触发时机

- “打开 App”定义为 App 进入前台，即 `MainActivity.onStart`。
- 每个本地日期最多尝试一次；同一天重复前台切换不重复上传。
- 若 App 进程未被杀死但跨天后重新进入前台，应允许新一天备份。

### FR-3 上传与保留策略

- 自动备份复用现有 DAV 导出逻辑，上传内容与手动“导出到 DAV”一致。
- 自动备份成功后清理远端旧自动备份，仅处理文件名以 `mewbook_auto_backup_` 开头且以 `.json` 结尾的文件。
- 按文件名排序保留最新 30 份自动备份，删除更旧文件；手动导出的默认文件名、自定义文件名与其他 JSON 文件不得删除。

### FR-4 状态与失败处理

- 自动备份失败不弹窗、不 Snackbar，不打断启动。
- DAV 设置页展示最近尝试时间、最近成功时间、最后错误或清理警告。
- 上传成功但清理旧备份失败时，备份仍视为成功，仅记录警告。

### FR-5 手动选择 DAV 备份导入

- 点击“从 DAV 导入”时，先列出远端目录中的 JSON 备份候选，包含自动备份、默认手动备份与自定义命名的手动导出。
- 用户手动选择一个备份后再进入导入预览。
- 确认导入时必须恢复用户刚选择的同一个备份文件，而不是重新选择最新文件。

### FR-6 手动导出文件名

- 点击“导出到 DAV”时，用户可以输入文件名；留空时使用默认 `mewbook_backup_yyyyMMdd_HHmmss.json`。
- 用户输入的文件名会去除路径分隔符与不安全字符，并自动补 `.json` 后缀。
- 手动导出的文件不纳入自动备份的 30 份保留清理范围。

## 非目标

- 不做自动恢复、冲突合并或双向同步。
- 不引入 WorkManager、后台定时任务、通知或新依赖。
- 不修改备份 JSON schema，不 bump Room 数据库版本。

## 验收标准

1. DAV 配置已保存且自动备份开启时，App 当天首次进入前台会上传一次备份。
2. 同一天再次进入前台不会重复上传。
3. 新的本地日期再次进入前台会触发新一次备份。
4. 自动备份成功后，远端只保留最新 30 个自动备份 JSON，手动导出文件不被自动清理。
5. 上传失败或清理失败只记录到 DAV 设置页状态，不打断用户启动。
6. DAV 导入时可以手动选择远端备份，预览与确认导入使用同一文件。
7. 手动导出到 DAV 时可以自定义文件名；留空使用默认文件名。
